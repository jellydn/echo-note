## Codebase Patterns
- First-launch setup: Track state in settings table with `first_launch_completed` key, check on App mount, render SetupWizard overlay
- Bundled resources: Place in `src-tauri/resources/`, configure in tauri.conf.json bundle.resources, access via `app_handle.path().resource_dir()`
- Privileged .pkg installation on macOS: Use `osascript` with `do shell script "..." with administrator privileges`
- Auto-install fallback chain: Try bundled → Homebrew → manual download page
- Use `sql<number>` template for aggregations (not applicable yet, but good to remember)
- Always use `IF NOT EXISTS` for migrations (already in practice)
